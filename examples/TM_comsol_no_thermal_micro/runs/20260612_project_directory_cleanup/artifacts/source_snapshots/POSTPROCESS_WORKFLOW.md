# TM Postprocess Workflow

## Normal training command

The standard D0020 route uses code defaults for the normal route:

```powershell
D:\anaconda3\envs\torch_env\python.exe main.py 8 400 23 TrainableReLU 3.0 --full --load-schedule-file load_schedule_D0020_extended.csv --run-suffix seed23_D0020
```

The command does not need `--coord-normalization unit_box`,
`--mixed-mechanics-mode history`, `--top-u-mode free`, or `--pff-model AT2`.
Those options remain exposed as advanced overrides for diagnostics.

By default, training writes checkpoints to `outputs/checkpoints/<short_run_id>/`,
results to `outputs/results/<short_run_id>/`, and logs to
`outputs/logs/<short_run_id>/`. Use `--output-root`, `--model-dir`,
`--result-dir`, `--log-dir`, `--figure-dir`, or `--curve-dir` only as advanced
path overrides.

## Automatic postprocessing

After training, `main.py` calls:

```python
run_results_postprocess(...)
```

from `postprocess_results.py`. The normal user-facing postprocess command is:

```powershell
D:\anaconda3\envs\torch_env\python.exe postprocess_results.py --model-dir <model_dir> --result-dir <result_dir>
```

Defaults:

- `out-dir = <result_dir>/curves`
- `figure-dir = <result_dir>/figures`
- figures are generated by default
- figure failures are non-fatal by default
- `--fail-on-figure-error` enables strict figure mode
- `--run-label` is optional; labels are shortened from seed/load tokens when omitted

`plot_results.py` is an optional direct plotting helper. It is not a required
second step for normal training results.

## Output files

The postprocess entry point writes functional filenames:

- `stress_strain_by_step.csv`
- `reaction_by_step.csv`
- `reaction_metric_availability.csv`
- `stress_strain_<short_label>.png`
- `reaction_strain_<short_label>.png`
- `final_fields_panel_<short_label>.png`

Output filenames should not use temporary repair labels.

The example root is source-only. Generated outputs belong under `outputs/`, and
audit/handoff packages belong under `runs/`.

## Reaction metric policy

`reaction_N_energy_exact` is the primary reaction when checkpoint exact reaction
is available. `reaction_N_energy_virtual_work` is a conjugacy check.
`reaction_N_legacy_top_sigma` and related boundary/cut values are diagnostics.

Old non-checkpointed runs cannot be declared no-softening cases from legacy top
reaction alone.

This workflow cleanup changes names, defaults, documentation, and postprocessing
entry points only. It does not change physics, material parameters, `l0`, TM
split formulas, history logic, alpha initialization behavior, or training losses.
